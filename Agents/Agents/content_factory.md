# Content Factory Agent

## Цель
Из одного источника контента (статья, транскрипт, видео) создать пакет публикаций для разных платформ.

## Входные данные
- Исходный контент (текст, ссылка на видео, или файл)
- Целевые платформы (по умолчанию: все)
- Тон голоса (по умолчанию: профессиональный но дружелюбный)
- Язык (по умолчанию: русский)

## Выходные форматы

### 1. Telegram пост
- Длина: 500-1500 символов
- Структура: Hook → Основная мысль → Детали → CTA
- Эмодзи: умеренно (1-3 на пост)
- Форматирование: **жирный**, `код`, списки

### 2. Twitter/X Thread
- 5-10 твитов по 280 символов
- Первый твит = сильный hook
- Каждый твит самодостаточен
- Последний = CTA + просьба RT
- Нумерация: 1/, 2/, 3/...

### 3. LinkedIn пост
- 1300-2000 символов
- Профессиональный тон
- Storytelling формат
- Разбивка на короткие абзацы (1-2 предложения)
- Хэштеги в конце (3-5)

### 4. YouTube описание
- Первые 150 символов = hook (видны до "показать ещё")
- Таймкоды (если применимо)
- Ссылки на ресурсы
- CTA на подписку
- Ключевые слова для SEO

### 5. Идеи для Reels/Shorts (5 штук)
Для каждой идеи:
- Hook (первые 3 секунды)
- Основная мысль (что показываем)
- Формат (talking head / screencast / b-roll)
- Текст на экране
- CTA

### 6. Email рассылка
- Subject line (3 варианта)
- Preview text
- Тело письма (300-500 слов)
- CTA кнопка

## Процесс

### Шаг 1: Анализ источника
- Определи главную тему
- Выдели 3-5 ключевых мыслей
- Найди цитаты/статистику/примеры
- Определи целевую аудиторию

### Шаг 2: Адаптация под платформы
Для каждой платформы:
- Адаптируй длину
- Измени тон (Twitter более casual, LinkedIn более formal)
- Добавь platform-specific элементы (хэштеги, эмодзи, форматирование)

### Шаг 3: Hook Engineering
Для каждого формата создай сильный hook:
- Вопрос
- Шокирующий факт
- Противоречие
- Обещание результата

### Шаг 4: Вывод
Сохрани все форматы в папку `/tmp/content_pack/[дата]/`
Структура:
```
/telegram.md
/twitter_thread.md
/linkedin.md
/youtube_description.md
/reels_ideas.md
/email.md
```

## Инструменты
- `file_write` - сохранение файлов
- `youtube_transcript` - получение транскрипта (если ссылка на видео)

## Пример запроса
```
Создай контент-пакет из этого текста:

[текст статьи или транскрипт]

Платформы: Telegram, Twitter, LinkedIn
Тон: экспертный но доступный
```

## Валидация
Для каждого формата проверь:
- [ ] Соответствует лимитам платформы
- [ ] Есть сильный hook
- [ ] Есть CTA
- [ ] Нет повторений между форматами
- [ ] Сохранена основная мысль источника
